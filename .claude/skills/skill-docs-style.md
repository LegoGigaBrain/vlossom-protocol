## Reference

## When to apply
- Writing or editing:
  - READMEs
  - How-to guides
  - Concept docs
  - Reference docs
  - Architecture docs
  - Changelogs
- Generating GitBook pages

## Instructions
1. Read `standards/docs/docs-style.md` before writing docs.
2. Apply:
   - clear headings
   - short paragraphs
   - skimmable lists
   - consistent terminology
   - cross-linking where useful
3. Tailor docs to:
   - audience (beginner, contributor, operator)
   - purpose (learn vs do vs look up).
