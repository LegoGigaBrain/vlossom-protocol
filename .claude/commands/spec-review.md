# spec-review

You are running the **Spec Review** workflow.

Goal:
Evaluate feature specifications for completeness, clarity, correctness, feasibility, and testability.

Primary Agent:
- @senior-architect

Supporting Agents:
- @backend-engineer
- @ux-product-strategist
- @design-reviewer
- @pragmatic-code-reviewer

Skills:
- Reviewer Voice
- Review Structure
- Review Dimensions
- S-Tier Design Principles (if UI)
- Code Style & API Standards
- Data Modelling Standards

---

## Steps

### 1. Clarify Scope
Ask for:
- Which spec(s) to review
- Stakeholders
- Intended timeline (MVP, alpha, beta)
- Constraints (tech, business, regulatory)

---

### 2. Pass 1 — High-Level Spec Validation
Check:
- Feature definition clarity
- User journey coherence
- Key scenarios covered
- Impact on architecture
- Ambiguities or missing sections

Output:
- Summary
- Strengths
- Primary concerns

---

### 3. Pass 2 — Detailed Spec Review
Check:
- Acceptance criteria clarity
- State transitions
- API definitions
- Data models
- Edge cases
- Performance considerations
- Loading/error/empty states
- Metrics and observability

For each issue:
- Title
- Severity
- Description
- Impact
- Recommendation

---

### 4. Review Dimensions
Score:
- Clarity
- Completeness
- Technical feasibility
- UX soundness
- Spec-testability
- Standards Alignment

---

### 5. Final Output
Use full review structure.
