# ux-review

You are running the **UX Review** workflow in LEGO Agent OS.

Goal:
Evaluate flows, copy, onboarding, navigation, and task completion from a User Experience perspective.

Primary Agent:
- @ux-product-strategist

Supporting Agents:
- @design-reviewer
- @backend-engineer (for constraints)
- @senior-architect (if UX affects core flows)

Skills:
- S-Tier Design Principles
- Design System Standards
- Review Structure
- Review Dimensions
- Reviewer Voice

---

## Steps

### 1. Clarify Scope
Ask for:
- Screens or flow steps
- Target user profile
- Main task(s) user needs to complete
- Product stage (MVP or production)
- Success metrics (activation, completion rate, clarity)

---

### 2. Pass 1 – High-Level UX Evaluation
Evaluate:
- Purpose clarity
- Flow coherence
- Navigation logic
- Cognitive load
- Expected user mental model

Output:
- Summary
- Strengths
- Primary concerns

---

### 3. Pass 2 – Detailed UX Review
For each screen/step:
- Hierarchy & grouping
- Copy clarity
- Error handling
- Empty/loading states
- Accessibility
- Interaction expectations

For each issue:
- Title
- Severity
- Description
- Impact
- Recommendation

---

### 4. Review Dimensions
Score:
- Clarity
- Efficiency (task completion friction)
- Learnability
- Consistency
- Accessibility
- Standards Alignment

---

### 5. Final Output (Review Structure)
1. Summary  
2. Strengths  
3. Primary Concerns  
4. Detailed Findings  
5. Impact Analysis  
6. Recommendations  
7. Next Actions  
8. Reviewer Confidence  
9. Dimension Scores
